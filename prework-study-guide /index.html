CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • All attributes in CSS follow the same pattern.
  • First the defining word at the start, telling the computer which elements are being effected.
  • Examples of this could be * for all or .card for card elements.
  • Then, within {} brackets, the attributes are listed.
  • It starts with a description of the attribute, such as width or color followed by a semi colon
  • Then it lists the choice peramiter, such as 80 px or blue, followed by a colon
  • Put together this looks like *{color:blue;width:80%;}
  • Once the file is done, the link is put in the HTML file.
  • an example of this would be link rel="stylesheet" href= file/location
  • Rel defines the kind of file being linked, href defines the reference path.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Issues are bits of code being worked on at the moment, what is in progress.
  • Issues use # signs for the headers, going from one meaning big, to two meaning smaller etc.
  • In generating a SSH key, you come up with a passkey, then take the url which follows afterwards and input it on github
  • git add -A commits all files in your current branch
  • git commit -m "text here" commits all of the code in that branch to your repository
  • git pull origin main pulls the code from the main branch into your current branch, making sure its up to date
  • git push origin feature/branch sends a request to your repository to merge this branch with the main branch
  • Once the code is in your git repository, you can make a pull request to merge the branch with the main code.
